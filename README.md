# wb-irrigationd

Сервис управления поливом на Wiren Board. Веб-интерфейс и API работают через FastAPI, данные хранятся в SQLite, реле управляются по MQTT.

```text
irrigationd/   приложение
packaging/     сборка и файлы пакета Debian
tests/         тесты
.github/       сборка релизов
```

## Установка пакета

Пакет предназначен для Wiren Board с архитектурой `arm64` и Python 3.9.2. Все зависимости входят в пакет, поэтому доступ к PyPI при установке не нужен.

```bash
sudo apt install ./wb-irrigationd_0.1.0_arm64.deb
sudo systemctl enable --now wb-irrigationd
sudo systemctl status wb-irrigationd
```

После запуска откройте:

```text
http://<IP-контроллера>/watering/
```

Основные пути:

```text
/etc/wb-irrigation/config.yaml             конфигурация
/var/lib/wb-irrigation/irrigation.db       база данных
/lib/systemd/system/wb-irrigationd.service служба systemd
/usr/share/doc/wb-irrigationd/examples/    примеры запросов
```

База и таблицы создаются при первом успешном запуске.

Журнал службы:

```bash
journalctl -u wb-irrigationd -f
```

## Настройка

Файл `/etc/wb-irrigation/config.yaml` содержит адрес MQTT-брокера, параметры веб-сервера, путь к базе данных и ограничения безопасности.

Веб-сервер по умолчанию слушает `127.0.0.1:8088`. Nginx Wiren Board публикует его по адресу `/watering/` и разрешает доступ только администраторам.

Проверка Nginx:

```bash
sudo nginx -t
ls -l /etc/nginx/includes/default.wb.d/watering.conf
curl -I http://127.0.0.1:8088/zones
```

## Зоны и расписания

Реле создаются в разделе «Настройки». Для каждого реле выберите устройство и канал либо задайте MQTT-топики вручную. Затем назначьте зоне одно или несколько реле. Все реле зоны включаются и отключаются одновременно.

Глобальный насос настраивается в том же разделе. При запуске зоны сервис сначала включает насос, ждёт заданную задержку и открывает реле зоны. При остановке насос отключается до закрытия реле. Если насос не настроен или отключён, зоны работают без него.

Расходомер передаёт текущий расход в литрах в минуту. Для защиты от сухого хода задаются минимальный расход, задержка срабатывания после запуска и тайм-аут данных. При недостаточном расходе или потере данных насос и зона останавливаются. Без расходомера насосом можно управлять, но полив по объёму недоступен.

При сохранении зоны сервис проверяет состояние и метаданные каждого реле. Проверка не переключает реле. Кнопка теста включает выбранное реле на одну секунду.

Датчик дождя один для всех зон. Значение `active_value` определяет состояние «идёт дождь». При дожде или отсутствии данных запуск блокируется. Если дождь начинается во время полива, насос и реле зоны останавливаются автоматически. Для выбранной зоны датчик можно отключить параметром `ignore_rain_sensor`.

Длительность полива, максимальное время и пауза между запусками задаются в минутах. Для полива по объёму укажите площадь зоны и норму воды в л/м². Целевой объём рассчитывается как площадь, умноженная на норму. Дни расписания выбираются флажками. Пересекающиеся активные расписания запрещены. Вкладка «Обзор» показывает ближайший запуск, состояние зон и датчика дождя, а также недельную диаграмму Ганта. Используется системное время контроллера.

Пример запуска зоны:

```bash
curl -X POST http://127.0.0.1:8088/api/zones/1/start \
  -H 'Content-Type: application/json' \
  -d '{"duration_min":1}'

curl -X POST http://127.0.0.1:8088/api/zones/1/stop
```

Другие примеры находятся в `/usr/share/doc/wb-irrigationd/examples/`. Документация API доступна по адресу `/watering/docs`.

## Запуск из исходного кода

```bash
python3 -m venv .venv
.venv/bin/pip install -e '.[dev]'
cp packaging/default-config.yaml config.yaml
```

Для запуска без Nginx задайте в `config.yaml` значение `web.root_path: ""` и доступный для записи путь `storage.path`, затем выполните:

```bash
.venv/bin/wb-irrigationd --config ./config.yaml
```

Интерфейс будет доступен по адресу `http://127.0.0.1:8088/zones`.

## Сборка пакета

Собирайте пакет в Debian Bullseye `arm64` с Python 3.9.2:

```bash
sudo apt update
sudo apt install python3 python3-venv dpkg-dev make
make deb
```

Результат: `dist/wb-irrigationd_<версия>_arm64.deb`.

Для сборки без доступа к PyPI:

```bash
make deb PIP_ARGS="--no-index --find-links=/path/to/wheelhouse"
```

## Публикация релиза

GitHub Actions собирает пакет по тегу `v<версия>`. Версия тега должна совпадать с `project.version` в `pyproject.toml`.

```bash
git tag -a v0.1.0 -m "wb-irrigationd 0.1.0"
git push origin v0.1.0
```

В релиз добавляются пакет и файл `SHA256SUMS`. Для повторной сборки запустите процесс `Публикация пакета Debian` вручную и укажите существующий тег.
